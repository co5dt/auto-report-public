### Changed

- Report narrative prompt strengthened with explicit Dutch language requirements to reduce hallucinated vocabulary and mixed-language output from smaller LLM models.
- `ReportSections.changeNarrative()` now always shows "Scripts (N)" and "iDoc templates (N)" lines even when counts are zero, ensuring file-type breakdown is complete.
- CLI runtime flow upgraded from placeholder message to full orchestration wiring all Phase 1ŌĆō6 components into a single executable command.
- Fat JAR packaging now excludes `META-INF` signature files (`.SF`, `.DSA`, `.RSA`) to prevent `SecurityException` from signed dependency JARs.
- Added `jackson-datatype-jsr310` dependency for `Instant` serialization in `--no-llm` JSON output.
- Added `-o/--output` option to control report output directory (default: current directory).
- Report output contract now uses `LlmProvider` for narrative generation with null-safe fallback, keeping `ReportGenerator` testable without external services.
- **Configurable agent count** ŌĆö Deliberation agent count is now configurable (1--3) via `CONSENSUS_AGENT_COUNT` env var, `consensus.agent.count` in `application.properties`, or code default (3). Single-agent mode returns unanimous result in round 1. `AppConfig` resolves settings with env > properties > default precedence. 13 `AppConfigTest` tests cover precedence, validation, and boundary values. 4 new `AgentDeliberationTest` tests cover single-agent, two-agent, and invalid-count paths.
- `RiskAssessment` output contract now includes consensus type, round reached, full vote list, and minority-opinion text required for downstream report transparency.
- Build configuration: Maven Surefire plugin configured with property-based JUnit 5 tag filtering. Default `mvn test` excludes `e2e`-tagged tests. Opt-in E2E profile (`mvn test -Pe2e`) runs only E2E-tagged tests for local Ollama integration testing.
- `ChangeData.isIDoc()` and `ChangeData.isScript()` made public for cross-package access by `ContextBuilder`.

### Fixed

- `AgentVote.parse()` now strips markdown code fences (` ```json ``` `) from LLM vote responses before JSON parsing. Ollama `qwen2.5-coder:7b` consistently wraps JSON in code fences, which caused 3/7 baseline pilot runs to fail with parse errors.
- Unknown LLM provider now produces actionable error message listing supported providers and returns exit code 2 instead of a generic stack trace.
- Invalid or missing roster file path now returns exit code 1 with a clear error message.
- Missing/sparse data in report inputs now produces explicit evidence-gap annotations rather than silent omission or empty sections.
- Deliberation failure paths now fail clearly on malformed/non-JSON vote payloads, missing prompt resources, and invalid vote schema values (missing fields, unsupported vote enum, out-of-range confidence).
